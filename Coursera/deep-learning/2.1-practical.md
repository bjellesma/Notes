# Setting Up your ML Apps

## Training Splits

With smaller datasets, it's typically OK to keep a ratio such as 80/10/10 for your training, dev (cross validation), and test sets but for larger datasets, it might be better to do 98/1/1. Cross validation sets are used to test the data in different models and use different hyperparams.

## Bias/Variance

Bias is error from overly simplistic assumptions in the model. A high-bias model underfits — it doesn't capture the underlying pattern in the data. Example: fitting a straight line to data that has a curve.
Variance is error from the model being too sensitive to the training data. A high-variance model overfits — it learns the noise along with the signal and fails to generalize. Example: a very deep polynomial that wiggles through every training point but performs poorly on new data.

<img width="864" height="390" alt="image" src="https://github.com/user-attachments/assets/2fd63486-c0c0-423d-8f43-06fb23f2ca69" />

An example of how to spot this is looking at a cat classification model. If the training set has a very low error at 1% but the dev set or test set has a high error at 11%, then we say that it has **high variance** because it has essentially overfit the data and is having a hard time on new data. If the training error and dev/test error are both high for a simple task (humans should be able to do with near perfect accuracy), then it has **high bias** and is underperforming.

<img width="911" height="453" alt="image" src="https://github.com/user-attachments/assets/a3c6da9e-0c77-4b8f-83e5-b924ca64d5fd" />

For high bias, possible solutions might be to get a bigger neural network or train for longer.
For high variance, possible solutions might be more data or getting regularization.

# Regularization

## L2 Regularization

As mentioned earlier, one solution to high variance can be regularization. Here we use **L2 Regularization** in our cost function . Note that this only applies to the weight as the bias is often just a number.

$$
J(w,b) = \frac{1}{m} \sum_{i=1}^{m} \mathcal{L}(\hat{y}^{(i)}, y^{(i)}) + \frac{\lambda}{2m} ||w||^2_2
$$

where

$$
||w||^2_2 = \sum_{j=1}^{n_x} w_j^2 = w^T w
$$

This works by discouraging large weights by adding a cost proportion to $\lambda$. A higher $\lambda$ pushes the weights toward zero, reducing variance at the risk of increasing bias.

In a neural network, this regularization param is updated to the following.

$$
||w^{[l]}||^2 = \sum_{i=1}^{n^{[l]}} \sum_{j=1}^{n^{[l-1]}} (w^{[l]}_{i,j})^2
$$

This matrix norm is known as the Frobenius norm. When used in regularization this way, the technique is called weight decay, because the penalty causes weights to shrink during training.

## Dropout

Dropout is a regularization technique where during each training iteration, neurons are randomly "dropped out" (deactivated) (In practice, this just means to zero out the neuron) with some probability, forcing the network to not rely too heavily on any single neuron.

How it works

* You set a keep probability p for each layer (e.g. 0.8 means 80% chance a neuron stays active)
* On each forward pass, a random subset of neurons is zeroed out
* The network sees a slightly different architecture every iteration
* At test time, dropout is turned off — all neurons are active

This is a good technique to reduce overfitting because the network can't learn to rely on certain neurons since any of them can be dropped out.

One drawback of dropout is that the cost function is no longer well defined.

    """
    Implements the backward propagation of our baseline model to which we added dropout.
    
    Arguments:
    X -- input dataset, of shape (2, number of examples)
    Y -- "true" labels vector, of shape (output size, number of examples)
    cache -- cache output from forward_propagation_with_dropout()
    keep_prob - probability of keeping a neuron active during drop-out, scalar
    
    Returns:

    m = X.shape[1]

## Early Stopping

The idea of early stopping is that you realize that error on both training and test are best at a certain iteration so that it's stopped early.

<img width="1972" height="999" alt="image" src="https://github.com/user-attachments/assets/29839f49-2dee-43ae-b81b-fde89b8e63e4" />

The downside of early stopping is that it couples optimization and regularization into a single decision. Stopping early prevents overfitting but also prevents the cost function from being fully minimized, making it harder to diagnose and fix each problem independently. This violates the idea of **orthogonalization** which is the idea that you should have seperate tools for seperate problems.

## Normalization

A highly used optimization technique is **normalization** which helps to get your inputs on the same scale. Say you have one feature on the scale from 1 to 1000 and another feature on the scale from 0 to 1, your data will ended up looking very elongated.

<img width="1995" height="1111" alt="image" src="https://github.com/user-attachments/assets/c9d32e37-74ef-4de7-9e54-dae9c2670b14" />

**1. Subtract the mean (zero-centering)**
$$\mu = \frac{1}{m} \sum_{i=1}^{m} x^{(i)}$$
$$x := x - \mu$$

**2. Normalize the variance**
$$\sigma^2 = \frac{1}{m} \sum_{i=1}^{m} (x^{(i)})^2$$
$$x := x / \sigma$$

## Vanishing/Exploding Gradients

In very deep networks, gradients can become extremely small or extremely large as they're backpropagated through many layers, making training very slow or unstable.

During backpropagation, gradients are multiplied by the weight matrices at each layer (Remember, this is backprop so it involves the chain rule of calculus). With L layers:

If weights are slightly greater than 1, gradients grow exponentially → exploding gradients
If weights are slightly less than 1, gradients shrink exponentially → vanishing gradients

For example, if every layer's weights are scaled by 1.5, by layer 150 the gradient is $1.5^{150}$ — astronomically large. If scaled by 0.5, the gradient is $0.5^{150}$ — essentially zero.

A partial solution to this is initializing the weights to better control their variance. This is known as the **Xavier/He initialization**. The 2 in the numerator is specific to a RELU activation, a 1 may be used with a TanH activation

$$
w^{[l]} = \text{np.random.randn(shape)} * \text{np.sqrt}\left(\frac{2}{n^{[l-1]}}\right)
$$

## Gradient Checking

Gradient checking is a debugging technique used to verify that your backpropagation implementation is correct by comparing your computed gradients against a numerical approximation.

The numerical approximation of a derivative uses the two-sided difference formula:

$$\frac{\partial J}{\partial \theta} \approx \frac{J(\theta + \epsilon) - J(\theta - \epsilon)}{2\epsilon}$$

This is more accurate than the one-sided version because the errors cancel out, giving an approximation on the order of $\epsilon^2$ rather than $\epsilon$.

### How it works

1. Flatten all parameters $w^{[1]}, b^{[1]}, \dots, w^{[L]}, b^{[L]}$ into a single vector $\theta$
2. Flatten all gradients into a single vector $d\theta$
3. For each parameter $\theta_i$, compute the numerical gradient using the formula above
4. Compare $d\theta$ (from backprop) against $d\theta_{approx}$ (numerical) using:

$$\frac{||d\theta - d\theta_{approx}||_2}{||d\theta||_2 + ||d\theta_{approx}||_2}$$

### Important caveats
- **Only use for debugging**, never in production training — it's far too slow
- Doesn't work with dropout because you're losing optimation of the cost function. The recomendation is to turn off dropout, perform grad check, and then turn back on dropout
- Run at random initialization, not after training. This is because the weights will have converged and may hide bugs.
